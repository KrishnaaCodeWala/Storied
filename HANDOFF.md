## Hard rules (never break)

1. Data-driven everything; content changes never require engine changes.
2. `npm test` green after every change; new features ship with tests.
3. Offline-first: playable from file:// and as standalone; network
   features degrade gracefully.
4. No copyrighted assets: no logos/character art/stills/audio/lyrics;
   quotes under 15 words; visuals are abstract SVG line-art (motifs.js).
   Wiki-sourced lore must carry CC BY-SA attribution + source URL.
5. After adding/renaming files: update sw.js FILES + bump VERSION,
   run `node build.js`, and re-sync Capacitor
   (`cp` changed files into ../storied-app/www + `npx cap sync android`).
6. Mobile-first: 48px+ targets, touch-action manipulation,
   reduced-motion respected.

## Gotchas discovered (save yourself the debugging)

- jsdom tests need `url: "https://storied.local/"` in the JSDOM
  constructor or localStorage throws (opaque origin).
- jsdom swallows event-handler exceptions; use a VirtualConsole when
  debugging silent failures.
- `.btn-guess` class is reused by the Study panel's "Build deck" button —
  always scope queries: `.answer-area .btn-guess`.
- localStorage keys keep the historical `quoted.` prefix on purpose
  (player-data continuity through the rebrand). Do not "fix" this.
- Top-level `const/let` in classic scripts share the global lexical
  scope across script tags — packstore.js declares the globals, game.js
  uses them. Tests eval all files concatenated in one `window.eval`.
- Pack order: strangerthings is intentionally first in the manifest
  (build.js `order`); tests assume it's the default selection.
- `window.Quoted` is kept as an alias of `window.Storied`.

- NEW GOTCHAS (both bit us this session):
  1. window.X assignments required for anything tests touch across
     eval boundaries — Studio and PackStore are now on window, same

     corrupts on "$&" sequences (packcheck's regex-escape literal
     re-inserted the matched script tag in build.js). ALL build.js
     replacements now use function replacements. Never revert.
